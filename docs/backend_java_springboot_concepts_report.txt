Java and Spring Boot Concepts Achieved In This Backend
======================================================

Project: retail-management-backend
Focus: Java, Spring Boot, Spring Security, JPA/Hibernate, scheduling, transactions, configuration, and SQL migration patterns implemented in this codebase.

1. Spring Boot Modular Architecture
- We implemented a modular backend structure where business domains are split into packages like sales, purchase, finance, tax, inventory, service, approval, and workflow.
- Examples:
  - src/main/java/com/retailmanagement/modules/erp/sales/service/ErpSalesService.java
  - src/main/java/com/retailmanagement/modules/erp/purchase/service/ErpPurchaseService.java
  - src/main/java/com/retailmanagement/modules/erp/finance/service/ErpFinanceService.java
  - src/main/java/com/retailmanagement/modules/erp/workflow/service/WorkflowTriggerService.java

2. Layered Architecture: Controller -> Service -> Repository
- Controllers expose APIs, services hold business logic, and repositories talk to the database.
- Examples:
  - src/main/java/com/retailmanagement/modules/erp/sales/controller/ErpSalesController.java
  - src/main/java/com/retailmanagement/modules/erp/sales/service/ErpSalesService.java
  - src/main/java/com/retailmanagement/modules/erp/sales/repository/SalesInvoiceRepository.java

3. Dependency Injection And Constructor-Based Wiring
- Spring-managed components are injected through constructors, often with Lombok @RequiredArgsConstructor.
- Example:
  - src/main/java/com/retailmanagement/modules/erp/foundation/controller/OrganizationController.java

4. JPA Entity Modeling
- Business tables are modeled as JPA entities using @Entity and @Table.
- Examples:
  - src/main/java/com/retailmanagement/modules/erp/sales/entity/SalesInvoice.java
  - src/main/java/com/retailmanagement/modules/erp/party/entity/Supplier.java

5. Spring Data JPA Repository Pattern
- Repository interfaces extend JpaRepository for CRUD and custom access patterns.
- Examples:
  - src/main/java/com/retailmanagement/modules/erp/finance/repository/AccountRepository.java
  - src/main/java/com/retailmanagement/modules/erp/finance/repository/VoucherRepository.java

6. Custom JPQL Queries
- We used @Query for reporting and custom data retrieval.
- Examples:
  - src/main/java/com/retailmanagement/modules/erp/expense/repository/ExpenseRepository.java
  - src/main/java/com/retailmanagement/modules/erp/tax/repository/TaxRegistrationRepository.java
  - src/main/java/com/retailmanagement/modules/notification/repository/NotificationRepository.java

7. Transaction Management
- @Transactional is used to keep ERP posting flows atomic.
- Examples:
  - src/main/java/com/retailmanagement/modules/erp/finance/service/ErpAccountingPostingService.java
  - src/main/java/com/retailmanagement/modules/erp/finance/service/ErpFinanceService.java

8. Split Posting Services For Complex Workflows
- Final side effects are isolated in dedicated posting services.
- Examples:
  - SalesInvoicePostingService.java
  - StockAdjustmentPostingService.java
  - StockTransferPostingService.java
  - ServiceReplacementPostingService.java
  - ReturnPostingService.java

9. DTO-Based API Contracts
- APIs use request/response DTOs instead of leaking JPA entities directly.
- Examples:
  - src/main/java/com/retailmanagement/modules/erp/sales/dto/ErpSalesDtos.java
  - src/main/java/com/retailmanagement/modules/erp/sales/dto/ErpSalesResponses.java
  - src/main/java/com/retailmanagement/modules/erp/finance/dto/ErpFinanceDtos.java
  - src/main/java/com/retailmanagement/modules/erp/party/dto/SupplierDtos.java

10. Spring Security Filter Chain Configuration
- Security is configured centrally with a SecurityFilterChain bean.
- Example:
  - src/main/java/com/retailmanagement/common/config/SecurityConfig.java

11. JWT Authentication And Request Filtering
- The backend uses JWT authentication with a custom filter and token provider.
- Examples:
  - src/main/java/com/retailmanagement/modules/auth/security/JwtAuthenticationFilter.java
  - src/main/java/com/retailmanagement/modules/auth/security/JwtTokenProvider.java

12. Custom UserDetailsService With Organization Awareness
- Authentication reloads users with organization context.
- Example:
  - src/main/java/com/retailmanagement/modules/auth/security/CustomUserDetailsService.java

13. Method-Level Authorization
- @PreAuthorize is used for module-specific permissions.
- Examples:
  - src/main/java/com/retailmanagement/modules/erp/sales/controller/ErpSalesController.java
  - src/main/java/com/retailmanagement/modules/erp/purchase/controller/ErpPurchaseController.java

- @RestControllerAdvice and @ExceptionHandler centralize API error responses.

Conclusion
----------
This backend demonstrates a real set of Java and Spring Boot concepts: layered architecture, dependency injection, JPA entities, repositories, JPQL, transactions, scheduled jobs, JWT security, method authorization, global exception handling, configuration classes, recurring processing, approval workflows, reporting services, and migration-driven SQL delivery.
